Phase 01 · Data Ingestion · One-time Setup

Step 01 · File Loader
Read Receipt Files
Load 100 raw .txt receipt files from receipt_samples/
Step 02 · Gemini Structured Output
Parse Receipts → Typed Objects
Gemini extracts merchant, date, items, totals, payment method, category into typed Receipt Pydantic models — handles all 9 receipt formats in a single prompt
Step 03 · Hybrid Chunker
Create Two Chunk Types per Receipt
Receipt-level (~100 docs) — full summaries for merchant/date/total queries
Item-level (~600 docs) — individual line items for product-specific queries
Step 04 · Gemini Embedding
Vectorize All Chunks
Embed ~720 documents using gemini-embedding-001 at 768 dimensions · batched in groups of 50 with exponential backoff retry
DB Write · Pinecone Index
Upsert Vectors + Metadata
~720 vectors stored with rich metadata: merchant, date (epoch), category, total, city, chunk_type, item_name, price · ID = MD5 hash of receipt_id:chunk_type:item
Phase 02 · Query Pipeline · Per User Request
User Types Question
Streamlit UI · app.py
Chat Interface
Thin client — sends POST /query via httpx · maintains conversation history in session state
Step 05 · FastAPI · api.py
POST /query Endpoint
Orchestration layer — receives QueryRequest, runs full pipeline, returns QueryResponse
Step 06 · Query Parser · Gemini
NL → Structured Filters
Extracts: date range, merchant, category, city, price range, aggregation type, chunk_type routing · sets needs_all_results=true for aggregation queries
Decision · Retrieval Strategy
Which Retrieval Mode?
Route based on query type and parsed filters
Semantic Query
Vector Search
Cosine similarity + metadata filters · returns top-k results
Aggregation Query
Metadata-Only
Dummy vector + pure filter — returns all matching records for full computation
Fuzzy / Mixed
Hybrid Search
Vector search + post-retrieval soft filters for fuzzy string matching
Step 07 · Retriever
Apply Soft Filters + Return Documents
Post-filter with fuzzy string matching on merchant/city · deduplicate by receipt_id for aggregations
Conditional · Aggregation Queries Only
Step 08 · Aggregator · Pure Python
Compute Aggregation
Calculates sum, avg, min, max, count, group-by — never delegated to LLM · anti-hallucination guarantee on all monetary values
Step 09 · Gemini 2.0 Flash
Generate Response
LLM formats retrieved docs + aggregation data into a 2–3 sentence answer · grounded only in retrieved context — never invents data
Step 10 · Streamlit UI
Display Results
Renders answer text + expandable receipt cards + aggregation breakdown table
